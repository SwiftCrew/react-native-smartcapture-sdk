# Contributing to `react-native-smartcapture-sdk`

First off — thank you for considering contributing! It's people like you that make open source great.

This document outlines the process for contributing to the project. Please take a moment to read it before opening an issue or pull request.

---

## Code of Conduct

Be respectful. We follow the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). Harassment, discrimination, or disrespectful behavior toward any contributor will not be tolerated.

---

## How Can I Contribute?

There are many ways to contribute:

- **Report bugs** — open a [GitHub issue](https://github.com/SwiftCrew/react-native-smartcapture-sdk/issues/new).
- **Suggest enhancements** — open a [feature request](https://github.com/SwiftCrew/react-native-smartcapture-sdk/issues/new).
- **Improve documentation** — even small typo fixes are welcome.
- **Submit pull requests** — see the workflow below.
- **Help triage issues** — comment on issues, reproduce bugs, suggest fixes.

---

## Reporting Issues

When opening an issue, please include:

- **Description** — clear and concise summary of the problem.
- **Steps to reproduce** — minimal reproducible example, ideally with code.
- **Expected vs. actual behavior**.
- **Environment**:
  - `react-native-smartcapture-sdk` version
  - `react-native` version
  - **Peer-dependency versions** (especially `react-native-vision-camera`, `react-native-reanimated`, `react-native-permissions`, `react-native-image-picker`, `@react-native-community/image-editor`)
  - iOS / Android version
  - Device / simulator (and Xcode / Android Studio version if relevant)
- **Logs / stack trace** — paste any relevant Metro / Xcode / `adb logcat` output.
- **Screenshots or screen recordings** — if it's a UI bug.

> Tip: run `npm ls react-native-vision-camera react-native-reanimated react-native-permissions` and paste the output — version mismatches are the #1 cause of "undefined" errors.

---

## Prerequisites

Before you start, make sure you have:

- **Node.js** ≥ 18
- **npm** ≥ 9 (or yarn ≥ 1.22)
- **Xcode** ≥ 15 (for iOS work) — including Command Line Tools
- **Android Studio** with an emulator or a physical device (for Android work)
- **CocoaPods** ≥ 1.12 (`sudo gem install cocoapods`)

## Development Setup

```bash
# Fork and clone the repo
git clone https://github.com/<your-username>/react-native-smartcapture-sdk.git
cd react-native-smartcapture-sdk

# Install dependencies
yarn install

# Build the library (produces lib/ via react-native-builder-bob)
yarn build

# Set up the example app
cd example
yarn install

# iOS
cd ios && pod install && cd ..
yarn ios

# Android
yarn android
```

You can also run example commands from the repo root:

```bash
yarn example:start    # start Metro for the example app
yarn example:ios      # run example on iOS simulator
yarn example:android  # run example on an Android device/emulator
```

## Pull Request Workflow

1. **Fork** the repository.
2. **Create a branch** from `main` with a descriptive name:

   ```bash
   git checkout -b feat/<short-description>
   git checkout -b fix/<short-description>
   ```

3. **Make your changes** following the coding guidelines below.
4. **Add or update tests** if your change affects behavior.
5. **Run quality checks** locally:

   ```bash
   yarn lint
   yarn typecheck
   yarn build
   ```

6. **Test on both platforms** — iOS and Android — using the example app.
7. **Update documentation** (`README.md`, `CHANGELOG.md`) if you've added/changed public API.
8. **Commit** using [Conventional Commits](https://www.conventionalcommits.org/):

   ```
   feat: add support for square crop ratio
   fix: prevent crash when image-editor returns null
   docs: clarify Reanimated 4 setup
   chore: bump react-native-vision-camera to 4.6
   ```

9. **Push** to your fork and **open a PR** against `main`. Fill in the PR template with:
   - **What** the change does.
   - **Why** it's needed.
   - **How** to test it (steps, screenshots / video for UI changes).
   - Linked issue, if any.

A maintainer will review and may request changes before merging.

## Coding Guidelines

### TypeScript

- Strict mode is enabled — no implicit `any`.
- Public API must be fully typed and exported from `src/index.ts`.
- Prefer `type` aliases over `interface` for public types unless extension is required.

### React / React Native

- Functional components + hooks only.
- Use `React.memo`, `useCallback`, and `useMemo` where it actually helps.
- Always clean up subscriptions, listeners, and timers in `useEffect` returns.
- Guard async work after unmount using a `mountedRef` or similar pattern.

### Style

- Follow the repository's ESLint + Prettier config — `yarn lint` must pass.
- 2-space indentation.
- Single quotes for strings; double quotes only inside JSX attributes when needed.
- Trailing commas in multiline arrays/objects.
- Keep components focused and small; extract reusable hooks into `src/hooks/`.

## Testing Changes

Before opening a PR, please verify:

- [ ] `yarn lint` passes.
- [ ] `yarn typecheck` passes.
- [ ] `yarn build` succeeds (output is generated under `lib/` by `react-native-builder-bob`).
- [ ] The example app launches **on iOS** and **on Android** without warnings or errors related to your change.
- [ ] Permission flows still work correctly (camera + gallery, granted / denied / blocked / limited iOS access).
- [ ] No regressions in:
  - Camera capture (front/back switch, 1x ↔ 2x zoom)
  - Gallery selection
  - Crop preview gestures (pan & pinch) for both `circle` and `rectangle`

## Releasing (maintainers only)

Releases are published from `main` after the version bump and CHANGELOG entry are merged.

```bash
# 1. update CHANGELOG.md with the new version + date
# 2. bump version (does not git-tag automatically)
yarn release:patch     # 1.0.0 -> 1.0.1
yarn release:minor     # 1.0.x -> 1.1.0
yarn release:major     # 1.x.x -> 2.0.0

# 3. publish to npm (this is what `release:*` runs at the end)
#    - prepublishOnly cleans, typechecks, and builds
#    - publishConfig.access = "public"
npm publish

# 4. tag and push
git add package.json CHANGELOG.md
git commit -m "chore: release vX.Y.Z"
git tag vX.Y.Z
git push origin main --tags
```

After publishing:

- Open a **GitHub Release** for the new tag and paste the relevant `CHANGELOG.md` section.
- Verify the package on npm: <https://www.npmjs.com/package/react-native-smartcapture-sdk>
